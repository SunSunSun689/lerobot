# 视频模糊原因分析

## 问题描述

采集的视频（`Arrange_flowers/data/episode_000000/videos/`）与标准数据集（`arrange_flowers-all`）相比，画面明显模糊。

## 对比数据

| 指标 | 采集视频 | 标准视频 |
|------|---------|---------|
| 分辨率 | 640×480 | 640×480 |
| 编码器 | h264 (libx264) | h264 (libx264) |
| 帧率 | 30 FPS | 30 FPS |
| **比特率 (arm)** | **638 Kbps** | **2210 Kbps** |
| **CRF** | **30** | **~23** |
| **GOP 大小** | **2** | **150** |
| 清晰度（Laplacian 方差） | 157 | 344 |

## 根本原因

### 1. CRF 值过高（主要原因）

CRF（Constant Rate Factor）是 H.264 编码的质量控制参数，**值越低质量越高，值越高压缩越重**。

- 采集视频使用 `CRF=30`（lerobot 框架默认值）
- 标准数据集使用 `CRF≈23`（ffmpeg 命令行默认值）
- 两者相差 7 档，导致采集视频比特率仅为标准的 **1/3**，画面细节大量丢失

CRF 参考范围：
- `0`：无损
- `18`：视觉无损（推荐高质量录制）
- `23`：ffmpeg 默认，质量良好
- `30`：明显压缩，细节模糊
- `51`：最低质量

### 2. GOP 过小（次要原因）

GOP（Group of Pictures）即关键帧间隔。

- 采集视频 `g=2`：每 2 帧一个 I 帧（关键帧）
- 标准视频 `g≈150`：每 150 帧一个 I 帧

lerobot 将 `g=2` 作为默认值，目的是支持训练时随机帧访问（任意帧附近都有关键帧，解码快）。但 GOP 过小会降低时间维度的压缩效率，在相同 CRF 下，大量比特率被分配给频繁的 I 帧，P 帧质量相对下降。

## 修复方案

### 修改内容

**1. `src/lerobot/datasets/lerobot_dataset.py`**

为 `_encode_video_worker` 和 `LeRobotDataset.__init__` 增加 `crf` 参数透传，使调用方可以自定义 CRF 值。

**2. `safe_record_lerobot_v2.py`**

创建数据集时指定 `crf=18`：

```python
dataset = LeRobotDataset.create(
    ...
    vcodec="h264",
    crf=18,       # 从默认 30 降低到 18，视觉无损级别
)
```

### 效果预期

| 参数 | 修改前 | 修改后 |
|------|--------|--------|
| CRF | 30 | 18 |
| 预期比特率 | ~638 Kbps | ~2000+ Kbps |
| 视觉质量 | 明显模糊 | 接近无损 |

## 相机硬件状态

三个 RealSense 相机的曝光和增益均为自动模式，硬件本身无异常：

| 相机 | 序列号 | 自动曝光 | 锐度 |
|------|--------|---------|------|
| top (D435I) | 406122070147 | 开启 | 50 |
| front (D435I) | 347622073355 | 开启 | 50 |
| wrist (D435) | 346522074669 | 开启 | 75 |

模糊问题完全由视频编码参数引起，与相机硬件无关。
